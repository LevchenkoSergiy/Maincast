1.Створити модель для зберігання інформації про турніри. Деталі: Назва турніру (string) Тип гри (enum: CS:GO, DOTA2, Valorant) Стадія турніру (string: Contenders, Main stage, Playoffs, Final) Prize pool (float) Live (boolean) Дата початку турніру (date) Дата завершення турніру (date) Опис турніру (text) Посилання на детальнішу інформацію (URL) 2.Створення таблиць бази даних Створити відповідну до моделі таблицю(і) турнірів в БД 3.Створити API для отримання та управління інформацією про турніри GET /api/tournaments: Отримати список всіх турнірів Відповідь API повертає список всіх турнірів з такими полями: Назва турніру Тип гри Стадія турніру Prize pool Live Дата початку турніру Дата завершення турніру Опис турніру Посилання на детальнішу інформацію GET /api/tournaments/{id}: Отримати інформацію про конкретний турнір за ID. Відповідь API повертає інформацію про турнір за вказаним ID з усіма необхідними полями. Валідація ID та обробка помилок (якщо турнір не знайдено). GET /api/tournaments/live: Отримати інформацію тільки про live турніри Відповідь API повертає список всіх live турнірів (де поле Live = true) з усіма необхідними полями. POST /api/tournaments: Додати новий турнір. Валідація даних, що надходять від клієнта (назва турніру, тип гри, стадія турніру, prize pool, live, дата початку та завершення, опис, посилання на детальнішу інформацію). Додавання нового турніру до бази даних. Відповідь API підтверджує успішне створення турніру. PUT /api/tournaments/{id}: Оновити інформацію про турнір. Валідація ID та даних, що надходять від клієнта. Оновлення інформації про турнір у базі даних за вказаним ID. Обробка помилок (якщо турнір не знайдено). Відповідь API підтверджує успішне оновлення турніру. DELETE /api/tournaments/{id}: Видалити турнір за ID. Валідація ID. Видалення турніру з бази даних за вказаним ID. Обробка помилок (якщо турнір не знайдено). Відповідь API підтверджує успішне видалення турніру.
